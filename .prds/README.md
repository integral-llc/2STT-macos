# PRDs

This directory contains Product Requirements Documents for the project.

## Structure

Each PRD lives in a numbered directory (e.g., `0001-short-slug/`) containing:

- **prd.md** - The requirements document
- **tasks.md** - Implementation task breakdown with checkboxes
- **decisions.md** - Running log of post-creation decisions and scope changes

## Statuses

| Status | Meaning |
|---|---|
| `draft` | Being written, not yet agreed upon |
| `ready` | Agreed upon, ready for implementation |
| `in-progress` | Actively being built |
| `done` | Fully implemented |
| `abandoned` | No longer relevant |

## Index

See [_index.md](./_index.md) for the full PRD registry.

## Portability

This directory is self-contained and can be copied into any repo to bootstrap the same PRD tracking system.
